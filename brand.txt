1. Brand Philosophy
TouchPulse sits at the intersection of navigation precision and AI intelligence. The design system must communicate three qualities simultaneously: serious technical credibility (like MagicLane or Naext), AI-forward modernity (like xAI or Vercel), and the warmth that makes the product feel approachable and human.

1.1  Brand Pillars
PILLAR 1
Precision
Every pixel earns its place. Navigation is a domain where accuracy is life-critical — the brand must project that same discipline. Thin lines, exact spacing, restrained colour.
PILLAR 2
Intelligence
TouchPulse is AI-native. The visual language should feel forward-looking without being sci-fi. Subtle aurora gradients, monospace code snippets, and crisp data visualisation signal intelligence without shouting it.
PILLAR 3
Momentum
Navigation is motion. The brand carries directional energy through arrow motifs, asymmetric layouts, and kinetic aurora effects.

2.2  Colour Hierarchy Rules
Teal and Gold must never appear at equal visual weight in the same section. One leads, the other disappears.

→  Use Teal for all interactive states — hover, focus, active, links, border highlights.
→  Use Gold for the single most important CTA on a page — "Get started", "Try free". Not for secondary actions.
→  Charcoal Black (#031119) is the canonical dark background. Never substitute #000000 — pure black reads as harsh.
→  Soft White (#F7F7F7) is the text colour on dark backgrounds. Pure white (#FFFFFF) is reserved for ghost button borders only.
→  Ink Blue (#1B354F) surfaces must always sit on top of Charcoal Black — never used as a standalone background on a light page.

2.4  Aurora Gradient System
The mesh/aurora gradient is the primary expressive element of the TouchPulse brand — used exclusively in hero sections, AI feature banners, and key landing transitions. It must never appear in functional UI (cards, tables, navigation).

On-brand aurora (Primary hero)
Technique: Stack 3 radial-gradient blobs on a #031119 base. Add noise grain texture at 5% opacity.
CSS implementation:
background: radial-gradient(ellipse at 20% 50%, rgba(1,180,175,0.55) 0%, transparent 60%),           radial-gradient(ellipse at 80% 30%, rgba(255,177,0,0.40) 0%, transparent 55%),           radial-gradient(ellipse at 50% 90%, rgba(27,53,79,0.60) 0%, transparent 70%);background-color: #031119;

AI feature aurora (Secondary)
Used for AI-specific sections — routing intelligence, prediction engine, live traffic AI. Uses blue-purple-teal rather than gold, signalling computation over energy.
background: radial-gradient(ellipse at 30% 60%, rgba(99,153,255,0.50) 0%, transparent 55%),           radial-gradient(ellipse at 70% 20%, rgba(180,120,255,0.45) 0%, transparent 60%),           radial-gradient(ellipse at 50% 100%, rgba(1,180,175,0.35) 0%, transparent 60%);background-color: #060c12;

✕  Use aurora gradients on cards, tables, nav bars, or any functional UI surface.
✕  Use more than two aurora variants on a single page.
✕  Animate the aurora on mobile — performance cost is not worth it.
✓  Add a subtle noise/grain layer at 5% opacity over every aurora background. This kills the digital smoothness and gives it depth.
✓  Let the aurora fade to transparent at the bottom — never hard-cut to the next section's background.

3.3  Colour Assignments for Text
→  Primary headings on dark: #F7F7F7 (Soft White)
→  Primary headings on light: #111827 (near-black, not pure black)
→  Body text on dark: rgba(247,247,247,0.75) — 75% Soft White

3.4  Typography Rules
✕  Use font-weight 700 (bold) for any heading — 500 (medium) is the maximum. Heavy weights look crude against the minimal layout.
✕  Center-align body text beyond a single line. Navigation documentation is never centered prose.
✕  Use all-caps on headings longer than 4 words. Eyebrows and labels only.
✕  Apply gradient text to more than one heading per page. If used, hero display only.
✕  Kern manually. Rely on letter-spacing: -0.02em to -0.03em on large headings only.
✓  Use sentence case everywhere — headings, nav links, button labels, badges.
✓  Set line-height: 1.1 on display and H1. line-height: 1.65 on body.
✓  Wrap long prose at 60–72 characters per line (max-width: ~640px on body text).

4. Layout & Spacing
Layout is where TouchPulse separates itself from generic SaaS. The reference brands (Vercel, xAI, SpaceX) share one trait: disciplined emptiness. Whitespace is not wasted space — it signals confidence.

4.1  Grid System
12-column grid. Max content width: 1280px. Gutter: 24px. Margin: clamp(24px, 5vw, 80px).

Column span conventions:
Full width (12 col): Hero sections, full-bleed media, navigation bar
10 col centred: Primary content sections with max-width prose
8 col centred: Documentation, blog posts, technical writing
6 col (2 × 6): Side-by-side feature comparisons, two-up cards
4 col (3 × 4): Feature grid (three columns), pricing tiers
3 col (4 × 3): Stats grid, partner logos, small card arrays

5. Component Library
Every component in the TouchPulse system follows the same principle: functional minimalism. No decorative shadows. No fills on buttons. Borders do the work.

5.1  Navigation Bar
Structure
Left: Logo wordmark (Inter 500, 18px, #F7F7F7 on dark / #031119 on light)
Centre: Navigation links — muted grey (rgba(247,247,247,0.55) on dark), 14px, with dropdown chevron "›" for items with sub-menus
Right: Ghost CTA button

Behaviour
→  Default state: transparent background, no border.
→  Scroll state (after 48px): background transitions to rgba(3,17,25,0.85) with backdrop-filter: blur(12px). 0.5px border on bottom at rgba(255,255,255,0.08).
→  Hover state on links: colour transitions from muted to #F7F7F7 over 150ms. No underline.
→  Active/current page: colour is #01B4AF (Electric Teal). No underline, no bold.

CSS spec (scroll state)
.nav {  background: rgba(3,17,25,0.85);  backdrop-filter: blur(12px);  -webkit-backdrop-filter: blur(12px);  border-bottom: 0.5px solid rgba(255,255,255,0.08);  transition: all 200ms ease;}
✕  Fill the nav bar with Ink Blue or any solid colour by default — it must be transparent on load.
✕  Use coloured nav links other than Teal for the active state.
✕  Add logo icons or illustrations next to the wordmark in the nav.

5.2  Button System
TouchPulse uses three button variants. Filled rectangular buttons are not part of the system.

Ghost box (Primary action)
An outlined box — transparent fill, 0.5px border, no border-radius beyond 6px. This is the default CTA style.
.btn-ghost {  padding: 8px 16px;  border: 0.5px solid rgba(255,255,255,0.7);  border-radius: 6px;  background: transparent;  color: #F7F7F7;  font-size: 13px;  font-weight: 500;  transition: background 150ms;}.btn-ghost:hover { background: rgba(255,255,255,0.06); }

Ghost box with Teal border (Secondary interactive)
Same as Ghost box but border-color: rgba(1,180,175,0.5) and color: #01B4AF. Used for documentation links, API reference CTAs.

Pill (Feature highlight / AI badge-button)
border-radius: 999px. Translucent colour fill. Used for AI feature actions and secondary nav pills.
.btn-pill-teal {  padding: 8px 18px;  border-radius: 999px;  background: rgba(1,180,175,0.12);  border: 0.5px solid rgba(1,180,175,0.40);  color: #01B4AF;  font-size: 13px;  font-weight: 500;}.btn-pill-gold {  background: rgba(255,177,0,0.12);  border: 0.5px solid rgba(255,177,0,0.45);  color: #FFB100;}

Icon arrow on buttons
→  Primary CTAs append an arrow: "Get started ↗" or "View docs →". Arrow is smaller (10px) and slightly muted vs button text.
✕  Use filled rectangular buttons with background-color on any CTA. This kills the minimal aesthetic immediately.
✕  Use more than one Gold button on a page. One signal, one place.

6. Iconography & Visual Motifs

6.1  Icon System
→  Icon library: Lucide (open source, consistent 1.5px stroke weight) or custom SVG at 1.5px stroke.
→  Icon sizes: 16px (inline/badge), 20px (button), 24px (standalone), 32px (feature highlight).
→  Icon colour: always inherits or uses the element's primary text colour. Never independently coloured icons unless in a teal or gold context.
→  Never fill icons. Stroke-based only — filled icons read as too casual.
→  Navigation arrows (→, ↗): Use Unicode or a thin custom SVG. These appear on nav links, buttons, and section transitions.

6.2  Arrow Motif
The arrow is TouchPulse's primary visual motif — it appears throughout the system as a signal of directionality, navigation, and forward momentum.
Navigation links: "Platform ›" — a single right-pointing chevron (›), 10px, muted
CTA buttons: "Get started ↗" — upper-right diagonal arrow, signals outbound/activation
Internal transitions: "Learn more →" — simple right arrow, stays within the experience
Data indicators: "↑ 18% this week" — directional deltas in metric cards

6.3  Grid & Dot Patterns
Subtle background patterns can appear in specific sections to add texture without complexity. These are always very low opacity (4–8%) and never compete with content.
Grid pattern: 1px lines at 40px intervals, rgba(255,255,255,0.04). For technical documentation sections.
Dot grid: 1px dots at 24px intervals, rgba(255,255,255,0.06). For dark hero sections.
Never combine aurora gradient AND a dot/grid pattern in the same section.

7. Motion & Animation
Animation at TouchPulse is purposeful. Every transition must communicate information or provide spatial orientation — never decoration for its own sake. Less than you think, slower than you want.

7.2  Easing
→  Use ease-out for elements entering the screen (they decelerate into position).
→  Use ease-in for elements leaving (they accelerate away).
→  Never use linear for UI transitions — it reads as mechanical.
→  Preferred easing: cubic-bezier(0.16, 1, 0.3, 1) — fast start, smooth landing.

7.3  Scroll Reveals
Content sections reveal on scroll using opacity + translateY:
.reveal {  opacity: 0;  transform: translateY(20px);  transition: opacity 500ms ease-out, transform 500ms ease-out;}.reveal.visible {  opacity: 1;  transform: translateY(0);}
✕  Animate cards horizontally — always vertical translateY reveals only.
✕  Stagger more than 5 items in sequence — the last item takes too long to appear.
✕  Use animation on reduced-motion users: always wrap in @media (prefers-reduced-motion: no-preference).
ALWAYS wrap animations in @media (prefers-reduced-motion: no-preference). This is both an accessibility requirement and a legal consideration in some markets.

10. Voice & Tone
Design and copy are inseparable. The visual minimalism only lands if the writing matches it. The brands TouchPulse references (xAI, Vercel, SpaceX) all write like engineers who have something important to say and no time to waste.

10.1  Writing Principles
→  Short. Then shorter. Read every line and cut the first word if it still makes sense.
→  Concrete over abstract. "Routes 2.4M journeys daily" beats "world-class scale."
→  Active voice. "TouchPulse calculates the fastest route" not "The fastest route is calculated."
→  No adverbs on claims. "incredibly accurate" → remove "incredibly." The claim speaks for itself or it doesn't.
→  Numbers are credibility. Always prefer a specific figure over a vague superlative.

10.2  Headline Patterns
Study the headline patterns from reference brands:
"Navigate smarter." — imperative, 2 words, full stop
"Real-time routing intelligence." — noun phrase, no verb needed
"Built for the speed of the city." — poetic constraint
"2.4M routes. Zero guesswork." — data + negation
"Your map, your rules." — possession + ownership

✕  World-class, industry-leading, cutting-edge, state-of-the-art
✕  Seamless, frictionless, intuitive, powerful, robust, scalable (without proof)

✕  Journey (in a metaphorical sense — TouchPulse is literally about journeys, use "trip" or "route")
✕  Disrupt, revolutionise, transform (unless a specific measurable change follows)

11.2  Core Accessibility Rules
→  All interactive elements must have a visible focus ring: outline: 2px solid #01B4AF; outline-offset: 2px.
→  All images require alt text. Decorative images: alt="".
→  Navigation must be keyboard-navigable. Tab order must follow visual order.
→  All form inputs require a visible label (not just placeholder text).
→  Motion: @media (prefers-reduced-motion) must disable or reduce all animations.
→  Touch targets: minimum 44×44px on mobile.
→  Error messages must not rely on colour alone — include an icon or text prefix.
